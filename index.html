Intro

formJS lets you validate and manage forms.
You can specify:

  • Validation rules ( also async ), errors and callback(s)
  • Custom CSS class(es) for dirty/error/valid fields
  • Events on which the validation will fire
  • How to collect data to send
  • AJAX call callbacks
  • and more...

Browsers Support

formJS is compatible with ( desktop & mobile versions ):

  • Chrome
  • Firefox
  • Internet Explorer 10+ / Edge *
  • Safari 9+

Usage

Include the plugin script in your page ( examples below from CDN ): 

<script src="https://unpkg.com/formjs-plugin/dist/formjs.min.js"></script>

<script src="https://cdn.jsdelivr.net/gh/simplysayhi/formJS/dist/formjs.min.js"></script>

or install via NPM: 

npm install formjs-plugin

and use as ES6 module: 

import Form from 'formjs-plugin';

or use as CommonJS module: 

var Form = require('formjs-plugin');

No dependencies.
This demo is developed with Bootstrap CSS but it is not a dependency.

HTML

This is the basic HTML structure to make the validation works.
( Bootstrap CSS classes are optional ) 

<form action="/mypath/mypage.php" method="POST" name="my-form-example" novalidate>

    <!-- BASIC REQUIRED TEXT INPUT -->
    <div data-formjs-question>
        <div class="answers-box form-group">
            <input name="myName" type="text" class="form-control" required />
            <label>Name</label>
        </div>
        <div class="field-error-message small">Write your name!</div>
    </div>
    
    <!-- REQUIRED FIELD WITH HTML STANDARD TYPE ( EMAIL ) -->
    <div data-formjs-question class="form-group">
        <div class="answers-box">
            <input name="myEmail" type="email" class="form-control" required />
            <label>Email</label>
        </div>
        <div class="field-error-message small">Check your email address, it is not valid!</div>
    </div>
    
    <!-- REQUIRED FIELD WITH SPECIAL TYPE:
         BASIC TYPE IS text
         SPECIAL TYPE ( NAMED SUBTYPE ) IS INSIDE data-subtype ATTRIBUTE -->
    <div data-formjs-question class="form-group">
        <div class="answers-box">
            <input name="myMobileNumber" type="text" data-subtype="mobile-number" class="form-control" required />
            <label>Mobile number</label>
        </div>
        <div class="field-error-message small">Check your mobile number, it is not valid!</div>
    </div>
    
    <!-- FIELD WITH SPECIAL TYPE AND VALIDATED ONLY IF FILLED:
         BASIC TYPE IS text
         SPECIAL TYPE ( NAMED SUBTYPE ) IS INSIDE data-subtype ATTRIBUTE
         FIELD VALUE IS EVALUATED ONLY IF FILLED ( SEE data-validate-if-filled ATTRIBUTE ) -->
    <div data-formjs-question class="form-group">
        <div class="answers-box">
            <input name="websiteUrl" type="text" data-subtype="url" class="form-control" data-validate-if-filled />
            <label>Website</label>
        </div>
        <div class="field-error-message small">Check your website url, it is not valid!</div>
    </div>
    
    <!-- BASIC TEXT INPUT NOT MANDATORY -->
    <div data-formjs-question>
        <div class="answers-box form-group">
            <input name="myInfo" type="text" class="form-control" />
            <label>Info</label>
        </div>
    </div>
    
    <button type="submit" class="btn btn-light">SEND</button>
    
</form>

Some notes about form / field attributes:

  • novalidate ( required ) enables JS field validation. It is set automatically if not present.

And some notes about field attributes:

  • data-formjs-question ( optional | empty attribute ) used for visual feedback on valid / invalid / dirty fields ( to be used on a container element for the field )
  • required ( required ) for fields that require an answer.
  • name ( required ) used inside "getFormData" method to collect form data.
  • type ( required ) used for field validation.
  • data-type ( optional ) by now only "number" is supported. See "strictHtmlValidation" option in Field Options section for details.
  • data-subtype ( optional ) for special field types ( eg: VAT number, fiscal code, etc - see the full list ).
  • data-exclude-data ( optional | empty attribute ) used to avoid the plugin to take in consideration the field when collecting form data ( used inside "getFormData" method ).

Initialization

Here an example of initialization: 

var formEl = document.querySelector('form');
var options = {...};
var formInstance = new Form( formEl, options );
formInstance.init().then(function( obj ){
    console.log('formInstance obj.instance', obj.instance);
    console.log('formInstance obj.fields', obj.fields);
});
  • formEl: ( required ) must be a single form node ( not a nodelist! ) or a CSS string selector var formEl = '.my-container .my-form';
  • options: ( optional ) a JS object with fieldOptions and formOptions ( see Options section )

formJS has some more properties:

  • isInitialized: whether or not the init method was called on the Form instance.
  • version: the version number.

Options:

formJS takes an object as second argument ( options ), as follow: 

{
    fieldOptions:   {...},
    formOptions:    {...}
}

Field Options

name
name, type
description 
beforeValidation
function / array of functions

Each function:

Fires just before a field is being validated and it is executed as Promise ( or chain/sequence of Promises if as array ).

Additionally, the "this" keyword is bound to the Form instance.

Arguments

  • fieldObj: ( Object ) 
    {
    fieldEl: fieldEl
}
    • fieldEl: JS node element ( the field that will be validated ) 
cssClasses
object

JS object with keys:

  • dirty: ( string ) CSS class(es) to be applied if the field "dirty" ( has some value )
  • error: ( string ) CSS class(es) to be applied if the field is not valid ( in any error case )
  • errorEmpty: ( string ) additional CSS class(es) to be applied if the field value is empty
  • errorRule: ( string ) additional CSS class(es) to be applied if the field value does not comply the validation rule
  • pending: ( string ) CSS class(es) to be applied when the field validation starts and removed when it finishes ( useful for async validations )
  • valid: ( string ) CSS class(es) to be applied if the field is valid

These classes are useful to show different error messages based on the error type and / or change the UI. 

focusOnRelated
boolean

Whether or not to focus on the related ( input / select ) field when selecting its radio answer 

handleFileUpload
boolean

Whether or not the plugin must handle the file upload. It is a simple file upload via ajax.
You must also set the enctype="multipart/form-data" attribute to the form.
( NOT used if ajaxSubmit: false ) 

handleValidation
boolean

Whether or not the plugin must handle the fields validation ( as single check and on form submit ). 

maxFileSize
number

Size limit ( in MB ) for each file to upload.
Set to 0 ( zero ) if you want to disable the size check.
Can also be set on the specific field via attribute data-max-file-size to set a custom limit. 

onPastePrevented
function / array of functions

Each function:

Fires when the paste event has been prevented on the field(s) ( see "preventPasteFields" option ).
( NOT fired if handleValidation: false )

Additionally, the "this" keyword is bound to the Form instance.

Arguments

  • fieldEl: JS node element
  • tempOptions: ( Object ) temporary options ( this object can be useful when manually validating a field with new options used only for that specific case ) 
onValidation
function / array of functions

Each function:

Fires just after a field has been validated.
If the field was evaluated as valid the entire form check is fired. So, this callback will run passing "fieldsList" as an array with one object for each form field.

Additionally, the "this" keyword is bound to the Form instance.

Arguments

  • fieldsList: ( Array ) list of JS objects*: 
    [{
    fieldEl: fieldEl,
    result: true,
    errors: {}
}]
    • fieldEl: JS node element
    • result: Boolean
    • errors: ( Object - only in case of invalid field )
  • tempOptions: ( Object ) temporary options ( this object can be useful when manually validating a field with new options used only for that specific case )

* in case of single field validation, as per method validateField, the array will contain only one object. In case of full form validation, as per method validateForm, the array will contain one object for each form field. 

onValidationCheckAll
boolean

If set to true, when a field is valid, this will fire the whole form validation without feedback on the UI. Also, the callback onValidation is fired passing all the validated form fields. 

preventPasteFields
string

CSS string selector for which the paste command will be disabled 

skipUIfeedback
boolean

If true, the css classes for error ( any type ) and valid fields will not be added ( for UI feedback as said by the name ) 

strictHtmlValidation
boolean

Activate JS controls for strict JS validation ( as for HTML5 attributes or emulation )

maxlength

emulation of working maxlength attribute: avoid to write more than specified length.

problem:
in some browsers, inputs with type="number" allow the user to insert non-numeric chars.

validateOnEvents
string

Events on which the validation will fire ( only for required fields )

Tip: always add "change" to the events in order to properly validate checkbox, radio, file inputs and selects.
For example: "change input" or "change blur".

( NOT used if handleValidation: false )

Default field options: 

{
    beforeValidation:       [beforeValidationDefault],
    cssClasses: {
        dirty:              'is-dirty',
        error:              'has-error',
        errorEmpty:         'has-error-empty',
        errorRule:          'has-error-rule',
        pending:            'is-pending',
        valid:              'is-valid'
    },
    focusOnRelated:         true,
    handleFileUpload:       true,
    handleValidation:       true,
    maxFileSize:            10,
    onPastePrevented:       [],
    onValidation:           [onValidationDefault],
    onValidationCheckAll:   true,
    preventPasteFields:     '[type="password"], [data-equal-to]',
    skipUIfeedback:         false,
    strictHtmlValidation:   true,
    validateOnEvents:       'input change'
}

validateField( field[, fieldOptions] )

Description:

Fires validation for the specified field.
Then, this also fires feedback on UI ( basic feedback with css classes ) and callback function(s) ( "onValidation" in "fieldOptions" ).

Parameters:

  • field: CSS string selector ( or node element * ) for the form field ( input, select, textarea )
  • fieldOptions: ( Optional ) JS object. For default values see Field Options section

* best to use a CSS string selector so that it looks for the field inside the instance form element.
Example:
 

formInstance.validateField('[name="myEmail"]');

Return value:

A Promise to which will be passed an object like: 

{
    fieldEl: element,
    result:  false,
    errors:  {...}
}
  • fieldEl: JS node element
  • result: ( Boolean ) if the field is valid or not
  • errors: ( Object ) only in case of invalid field 
validateForm( [fieldOptions] )

Description:

Fires validation for the form.
Then, this also fires feedback on UI for each field ( basic feedback with css classes ) and callback function(s) passing the full array of fields ( "onValidation" in "fieldOptions" ).

Parameters:

  • fieldOptions: ( Optional ) JS object. For default values see Field Options section

Return value:

A Promise to which will be passed an object like: 

{
    fields: [{...}],
    result: false
}
  • fields: ( Array of Objects ) a list of the form fields as returned by "validateField" method
  • result: ( Boolean ) check for the form, if it is valid or not

Prototype Methods

You can use these methods ONLY from the original Form object ( not from a new instance - but you can always set each single property/method from the class prototype ):

name, parameters
parameters
notes 
addValidationErrors( newErrors )

Parameters:

  • newErrors: JS object containing the new validation error(s) to add ( see example below ).

The key / value pairs must be passed as:

  • key: the same used as data-subtype ( or type ) attribute, without dash and with Capital letter instead.
    Eg: to validate "mobile-number" ( as data-subtype ) the rule is "mobileNumber"
  • value: a function with these arguments:
    • string: the string the user put in the field
    It must return a JS object with all the specific errors.

Notes:

This will affect both old and new instances.
Can also be used to override existing validation errors. 
addValidationRules( newRules )

Parameters:

  • newRules: JS object containing the new validation rule(s) to add ( see example below ).

The key / value pairs must be passed as:

  • key: the same used as data-subtype ( or type ) attribute, without dash and with Capital letter instead.
    Eg: to validate "mobile-number" ( as data-subtype ) the rule is "mobileNumber"
  • value: a function with these arguments:
    • string: the string the user put in the field
    • fiedlEl: the JS node element of the field
    It must return a JS object ( at least ) like: 
    {
    result: true
}
    In case of invalid field you can add the "errors" key to return the specific errors.

Notes:

This will affect both old and new instances.
Can also be used to override existing validation rules. 
setOptions( options )

Parameters:

  • options: JS object with the fieldOptions/formOptions to override ( see Options section )

Notes:

This will affect ONLY new instances

Examples of usage: 

Form.addValidationErrors({
    username: function ( string ) {
        var obj = {};
        var strLength = string.length;
        if( strLength < 3 ){ obj.minlength = true; }
        if( strLength > 24 ){ obj.maxlength = true; }
        if( /[^\w\-\.\@]/.test(string) ){ obj.invalidChars = true; }
        if( !/^[\w]/.test(string) ){ obj.invalidStartChar = true; }
        return obj;
    }
});

Form.addValidationRules({
    myPassword: function( string, fieldEl ){
        var obj = {
            result: /^(?=.*[0-9])(?=.*[a-z])(?=.*[A-Z])[0-9a-zA-Z]{8,}$/.test(string)
        };
        // ERRORS CAN BE ATTACHED HERE BUT IT'S BETTER TO USE METHOD addValidationErrors
        if( !obj.result ){
            if( string.length < 8 ){ obj.errors.minlength = true; }
        }
        return obj;
    }
});

Form.setOptions( { fieldOptions: {maxFileSize: 100} } );

Or directly put it in the prototype: 

// ADD/OVERRIDE A VALIDATION ERROR METHOD
Form.prototype.validationErrors.email = function( string ){ ... }

// ADD/OVERRIDE A VALIDATION METHOD
Form.prototype.validationRules.myPassword = function( string, fieldEl ){ ... }

// OVERRIDE AN OPTION
Form.prototype.options.fieldOptions.maxFileSize = 100;

Or directly put it in the specific Form instance ( for example to update an option ): 

// OVERRIDE AN OPTION
formInstance.options.fieldOptions.maxFileSize = 50;

You can also add / override validation rules/errors from the Form instance but it doesn't have much sense.

Built-in Errors

After validation, in case of invalid field, the validation method will return the "errors" object.

In case of empty field value, "errors" object will return only the "empty" error like: 

{
    empty: true
}

In case of invalid field value ( not compliant with validation rule ), "errors" object will return "rule" error and more specific errors as described below: 

{
    rule: true,
    ...
}
